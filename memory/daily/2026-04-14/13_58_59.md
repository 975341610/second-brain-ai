### 🛠️ 修复摘要与关键实现点：

1. **块手柄一拖动就弹菜单导致无法拖动**
   - **根因**：拖拽（Drag）和点击（Click/MouseDown）事件在手柄元素上发生了冲突。鼠标按下准备拖拽时，触发了打开菜单的逻辑，弹出的菜单遮挡或打断了原生的拖拽行为。
   - **修复**：我在 `NovaBlockEditor.tsx` 中的 `DragHandle` 增加了 `onDragStart` 事件拦截，当开始拖拽时，强制调用 `setIsBlockMenuOpen(false)` 将块菜单关闭，确保拖拽过程丝滑无阻。

2. **拖动指示线（黑线）显示 Bug 且拖拽后不消失**
   - **根因**：Tiptap 自带的 Drop Cursor 插件在某些复杂的 React 渲染或拖拽中断（比如松开鼠标时的边界情况）下，没能正确收到 `dragend` 或 `drop` 事件，导致代表黑线的 DOM 节点残留在页面上。
   - **修复**：我在拖拽结束的钩子（`handleDragEnd`）中增加了一层兜底清理逻辑。只要拖拽一结束，无论正常 drop 还是意外中断，都会主动查找并移除残留在 DOM 树里的 `.prosemirror-dropcursor-block` 元素。

3. **错别字采纳后红色波浪线不消失**
   - **根因**：`AISpellcheck.ts` 在执行“一键采纳”建议时，只是通过 `tr.insertText` 替换了底层的文字。但是 ProseMirror 的 `DecorationSet`（负责画那条红波浪线）并没有被明确告知这个错误已经被解决，由于文字长度可能变化，波浪线甚至会错位残留。
   - **修复**：我在 `applySuggestion` 执行文字替换时，往事务里塞入了一个专属的标记 `tr.setMeta('spellcheckResolved', { from, to })`。然后在其对应的 Plugin State 的 `apply` 方法里拦截这个标记，精准定位并从集合中彻底 `remove` 掉那条对应的 Decoration（波浪线），做到药到病除。
