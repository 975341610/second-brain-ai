**这是一个非常经典的 React + Tiptap（富文本编辑器）的冲突血案：**
当你把背景纸切到 `none`（无）的时候，我的代码里写了 `return null`，这会导致背景纸的 DOM 节点直接从网页上**被拔掉**。
偏偏这个背景纸和 Tiptap 编辑器是“好邻居”（兄弟节点）。背景纸一消失，编辑器的节点位置就往前挪了。而 Tiptap 底层是自己偷偷控制 DOM 的，等 React 下次刷新时，一核对位置发现“哎？邻居怎么没了，位置全乱了！”，直接底层逻辑崩溃，抛出 React 渲染死循环报错，然后就白屏了。

**怎么修：**
不能用 `return null` 粗暴拔掉它！我们要**“占着茅坑不拉屎”**——让这个背景纸节点永远留在那里，当选 `none` 的时候，我只把它的透明度（opacity）变成 0，让它彻底隐身，但节点还在。这样 React 和 Tiptap 就能和平共处了！
