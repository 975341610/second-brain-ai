你跟着我一步一步点，就能完美回到 `ea215b6`（提交信息应该是：`fix: 修复 TableOfContents...`）这个干净的版本：

### 第一步：让时间线倒流 (Reset)
1. 在 GitHub Desktop 左侧，点击 **History（历史记录）** 标签页。
2. 往下翻，找到你要回去的那个提交：`fix: 修复 TableOfContents React key 重复报错并升级版本至 v0.16.3`。
3. **鼠标右键**点击这条记录，在弹出的菜单里选择 **Reset to commit（重置到此提交）**。

*(点完之后，你的版本已经回去了，但是！GitHub Desktop 为了保护你的代码，会把之后你写的那些“不要了的代码”全都塞回到你的“暂存区”里。)*

### 第二步：彻底扔掉不要的代码 (Discard)
1. 点击左侧回到 **Changes（更改）** 标签页。
2. 你会看到一堆被修改的文件（也就是咱们不要的那些代码）。
3. 找到列表右上角（或者直接右键全选所有文件），点击带垃圾桶图标的 **Discard all changes（放弃所有更改 / 撤销）**。
4. 弹窗确认，点击 **Discard（放弃）**。

✨ **搞定！** 现在你电脑上的文件夹里，代码已经彻彻底底、干干净净地回到 `ea215b6` 啦！
