在 `nova_repo` 目录下，修改 `nova-block/src/components/widgets/HabitTrackerComponent.tsx`，实现以下两个核心目标：

**1. 支持上传自定义图标 (Custom Icon Upload)**
- 当前习惯的 `icon` 只是纯文本 Emoji。需要支持用户上传本地图片作为打卡图标。
- **设置面板改造**：在 `isSettingOpen` 展开的习惯列表里，给图标编辑区旁边加一个隐藏的 `<input type="file" accept="image/*" />` 和一个上传按钮（比如替换原有的小 icon，或者加个相机的小 icon）。
- **图片转 Base64**：用户选择图片后，使用 `FileReader.readAsDataURL()` 将其转换为 Base64 字符串，并通过 `updateHabit(h.id, { icon: base64String })` 保存。为防止 localStorage 爆满，可以对超大图片做一下简单的 canvas 压缩，或者限制宽高（可选但推荐）。
- **渲染兼容**：在所有显示 `icon` 的地方（组件左上角头部、设置面板列表、日历格子中心），判断如果 `icon` 以 `data:image/` 或 `http` 开头，则渲染 `<img src={icon} className="object-contain w-full h-full" draggable={false} />`；否则正常渲染为文本。
- **保留渐进特效**：日历格子中底层的灰度轮廓和上层用 `clip-path: inset(...)` 显形的特效，必须同样完美作用于这个 `<img>` 标签上。

**2. 深度性能优化 (解决掉帧卡顿)**
- **问题根源**：当前日历有 35~42 个格子，每次点击打卡，触发 `logs` 状态更新，导致所有格子全部重新渲染，且每个格子都在跑 `logs.find()`（O(N) 复杂度），造成严重掉帧。
- **优化点 A: React.memo()**：将单个日历格子抽离为独立的 `<HabitCell />` 组件，并用 `React.memo` 包裹。只传入必要的标量 props（`dateStr`, `val`, `target`, `color`, `icon`, `isToday`, `isCurrMonth`, `onClick`, `onContextMenu` 等）。
- **优化点 B: Map 查表**：在主组件中，使用 `useMemo` 将当前 `activeHabit` 的所有 `logs` 映射为一个 `logsMap` (如 `Record<string, number>`)。渲染 `<HabitCell>` 时，直接从 `logsMap[dateStr] || 0` 取值，将 O(N) 搜索降至 O(1)。
- **优化点 C: useCallback**：确保传给 `HabitCell` 的 `onClick` 和 `onContextMenu` 等回调函数被妥善稳定（或者直接在 `HabitCell` 内仅抛出 `dateStr`，主组件里的 handler 用 `useCallback` 包裹）。

**保留现有风格**：绝对不可破坏现有的手账风/新粗野主义（Neo-brutalist）的纸张背景、粗边框、硬阴影、手写字体以及满级纸屑动效。
