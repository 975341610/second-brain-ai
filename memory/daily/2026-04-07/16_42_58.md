**1. 刷新清空问题 (Local Storage 覆盖 BUG)**
- `HabitContext.tsx` 中的 `logs` 和 `habits` 在初始化时由于 `useEffect` 的执行顺序问题，会被默认的空数组 `[]` 覆盖掉 `localStorage` 里的真实数据。
- **修复方案**：将 `useState` 改为惰性初始化 (Lazy Initialization)，如 `useState(() => { const saved = localStorage.getItem(...); return saved ? JSON.parse(...) : default; })`，并移除旧的冗余 useEffect 加载逻辑。

**2. 无法切换不同的打卡**
- `HabitTrackerComponent.tsx` 中的 `<select>` 使用了 `appearance-none` 且没有提供下拉箭头，导致用户以为无法点击或难以交互。
- **修复方案**：在 select 旁边添加一个小下拉箭头 (`ChevronDown` 图标)，使其看起来像个真正的下拉菜单；或者直接改用更明显的 UI 切换方式。

**3. 新建打卡无法改名 & 无法设置目标次数**
- **修复方案**：在 `HabitTrackerComponent.tsx` 的快速设置面板 (`isSettingOpen`) 中，将单纯的文本展示 (`h.name`) 替换为可编辑的 `<input>`，允许用户实时修改打卡的 `name`（名称）、`icon`（图标）、`targetValue`（目标点击次数）和 `color`（颜色，可用原生 `<input type="color">`）。输入变化时调用 `updateHabit(h.id, { ... })`。

**4. 单格点击逻辑与百分比显示**
- 当前逻辑是 `(currentValue + 1) % (targetValue + 1)`，导致点满后多点一下直接归零，容易误触。
- **修复方案**：
  - 左键点击：只增不减，最高累加到 `targetValue` 封顶，不再自动归零。
  - 右键点击 (`onContextMenu={... e.preventDefault()}`)：减少次数或归零，方便撤销误触。
  - 进度显示：当该习惯的 `targetValue > 1` 时，在日历格子里面除了显示日期，再用更小的字号（或进度条百分比文字）清晰地显示如 `3/8` 的进度标识，让用户明确知道还差几次点满。
