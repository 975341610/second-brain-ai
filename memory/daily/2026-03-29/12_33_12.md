这次的问题出在 **Build React Frontend** 这一步的 `npm install` 阶段。
你看日志里有一句非常诡异的报错：`npm error Exit handler never called!`
这通常是 Windows 上面 Node/NPM 经常抽风的一个 Bug（NPM 直接崩溃了，包根本没有安装完）。

更离谱的是，即使 NPM 崩溃了，因为我们用的命令是 `npm install && npm run build`，它竟然没有终止，反而继续去执行了打包命令（`npm run build`）。结果因为前面的依赖包根本没装上，自然就报错说找不到 `vite` 啦！

**解决办法非常简单：**
1. 我把 `npm install` 换成了在 CI 环境中更稳定、更不容易报错的 **`npm ci`**。
2. 我把连续执行的 `&&` 拆成了两行命令分别执行，这样就不会再出现前面崩了后面还在跑的灵异事件了。
