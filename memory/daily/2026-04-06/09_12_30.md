# AI 行业晨报 (2026-04-06)

*   **Anthropic 调整政策**：Claude 订阅不再覆盖第三方智能体框架（如 **OpenClaw**）的使用量，引发行业关注。

*   **微软推进智能体化**：微软发布 **Copilot Cowork** 预览版，引入多模型协作（传闻包含 GPT 与 Claude），旨在执行更复杂的自动化任务。

*   **OpenAI 动态谨慎**：在指定时间范围内，未发现 OpenAI 发布重大产品或模型更新。

### 🤖 OpenAI & Claude 动态

*   **Anthropic 限制第三方工具使用**：自 2026 年 4 月 4 日起，Anthropic 宣布其 Claude Pro/Max 订阅将不再覆盖通过 OpenClaw 等第三方智能体框架产生的使用量。用户如需继续使用，必须启用“超额使用”（Extra Usage）选项并单独付费。此举被认为是由于智能体工作流对计算资源造成了巨大压力。对于严重依赖 OpenClaw 等框架进行自动化或开发的用户，成本将显著增加。
*   **OpenClaw 创始人加入 OpenAI**：在 Anthropic 调整政策之际，OpenClaw 的创建者 Peter Steinberger 宣布加入 OpenAI，以推动“下一代个人智能体”的发展。OpenClaw 项目将作为一个开源基金会项目继续存在，并得到 OpenAI 的支持。
*   **OpenAI 暂无重大更新**：在本次观察的时间窗口内，未检索到 OpenAI 官方发布关于模型、API 或产品的重大公告。

### 💻 Cline 观察

*   **信息有限，谨慎关注**：Cline（前身为 Claude Dev）被描述为一个“智能体编码助理”，能够执行文件编辑、终端命令等多步任务。目前关于其最新动态的信息较少，主要在一个名为 `9router` 的多 AI 工具连接器中被提及，表明其可作为后端模型之一被调用。对于寻求高级代码生成与自动化能力的开发者，Cline 仍是一个值得关注的潜在工具，主要适用于 VS Code 等开发环境。

### 🌐 AI Agent (智能体) 趋势

*   **平台与开源框架的博弈加剧**：Anthropic 对 OpenClaw 的政策调整，凸显了中心化 AI 平台与快速发展的开源智能体生态之间的紧张关系。平台方出于成本与控制的考量，开始收紧对高强度自动化使用的免费或低价访问权限。
*   **巨头加码智能体工作流**：微软推出的 Copilot Cowork Frontier 预览版，旨在让 AI 从被动的问答助手转变为能主动执行多步复杂任务的“数字同事”。这标志着行业巨头正将战略重心转向更具自主性的智能体应用，以抢占企业级市场。
*   **多模型协作成为新方向**：据报道，微软的 Copilot Cowork 内部采用了多模型协作机制，例如使用 GPT-4 起草内容，再由 Claude 模型进行审阅和批判。这种“团队协作”模式有望提升 AI 生成内容的准确性和可靠性，成为未来智能体架构的主流范式。

### 🔍 其他扩展观察

*   **Google Gemini 调整定价并深化集成**：Google 为 Gemini API 引入了更灵活的定价层级（标准、灵活、优先），以适应不同的工作负载和成本需求。同时，Gemini 也开始更深入地集成到 Google 地图等核心应用中。
*   **Microsoft Copilot 条款引争议**：微软的 Copilot 服务条款中一条将产品定义为“仅供娱乐”的旧条款在近期被广泛关注，微软发言人回应称，这是“遗留语言”，将会在下次更新中修改。同时，Windows 11 的新版 Copilot 正在从原生应用转向基于 Web 组件（Edge 内核）的 PWA 应用，导致其内存占用有所增加。
