Agent: 我已经将这 5 个问题针对 `nova-block` 工程进行了全面修复，这次完全定位到了问题的本质原因（涉及 Tiptap NodeView 和 React 渲染树的底层机制冲突），修复详情如下：

1. **代码块输入后消失**：
   - 原因：Tiptap 官方要求使用 `ReactNodeView` 渲染 `CodeBlock` 时，`NodeViewContent` 必须精确地作为 `code` 标签呈现。之前嵌套了一层原生的 `<code>` 导致 ProseMirror 无法匹配 DOM，从而在文档状态更新（如打字）时直接销毁节点。
   - 修复：重构为 `<NodeViewContent as="code" ... />`，完美符合 Schema 规范，输入不会再消失。

2. **便利贴输入后消失与拖拽异常**：
   - 原因：同样是 DOM 结构匹配问题。`StickyNote` 在扩展配置中返回的 `renderHTML` 是纯 `div`，但在 React 组件中却嵌套了三层 `div`，导致打字时 Tiptap 认为节点非法而清除。
   - 修复：同步了扩展的 `renderHTML` 和 `StickyNoteComponent` 的层级结构，并为 `NodeViewContent` 指定了明确的 `as="div"`。

3. **注脚不显示（点击无反应 / 被吞）**：
   - 原因：注脚 `footnote` 是一个 `inline` 行内节点。先前的弹窗面板使用了 `<div>`（块级元素）。在 HTML 规范和 ProseMirror 机制中，行内元素包含块级元素会导致 DOM 被浏览器强行拆分，从而破坏 NodeView 渲染树。
   - 修复：将 `FootnoteComponent` 内所有的 `div` 容器替换为 `span` (结合 flex/block 布局保留样式)，消除渲染崩溃。

4. **TOC（目录）点击无跳转**：
   - 原因：Tiptap 之前生成 Heading ID 的正则表达式 `[^\w\s-]` 剥离了所有的**中文字符**，导致生成的锚点 ID 全部变成了乱码或降级为随机字符串，而 TOC 侧边栏读取不到正确的 ID。
   - 修复：更新正则以支持中文字符保留（`[^\u4e00-\u9fa5a-zA-Z0-9-]`），锚点和目录 ID 完美对齐。

5. **引用 (Blockquote) 样式失效**：
   - 原因：受到 Tailwind Typography 插件的 `.prose` 全局样式优先级覆盖（强制加上了背景色和引用的伪元素引号）。
   - 修复：在 `novablock-core.css` 中增加了更高优先级的 `.novablock-editor blockquote` 选择器进行强覆盖，彻底清除了不需要的引述引号及样式。
