- **删除失败**：提示“删除失败”说明前端其实把 ID 传给后端了，但后端 `DELETE /api/notes/{id}` 接口拒绝了，或者返回了错误。有可能是因为子页面身上挂着某些外键约束，或者后端的级联删除（CASCADE）没写对。
- **父页面变成新笔记**：这是前端的“状态错乱（Draft Promotion Bug 的变种）”。当你试图删除子页面时，前端可能清空了当前高亮的 `activeNote`，并强制新建了一个空草稿，然后把这个空草稿的状态**错误地**绑到了你切回来的父页面上，导致父页面内容被重置或覆盖了！

### 问题 3：代码块高亮死活不出来
截图里的代码确实还是死板的单色（全是品红/青色）。这意味着 Tiptap 的 `CodeBlockLowlight` 插件压根没把这些代码解析成带着 `.hljs-keyword` 这种 class 的 span，或者我们在 CSS 里写的选择器依然没打过某个不知名的 Tailwind 类。
