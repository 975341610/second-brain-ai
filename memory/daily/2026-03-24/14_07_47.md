### 🛠️ 修复与优化内容

1.  **新建笔记保存异常（刷新/闪烁）**：
    *   **原因定位**：之前 `NotionEditor` 的编辑器实例绑定了 `note.id` 作为 Key，导致草稿转正（ID 从负数变正数）时编辑器会被彻底销毁重建，造成焦点丢失和页面闪烁。
    *   **修复方案**：解除了 `useEditor` 对 `note.id` 的强制依赖。现在 ID 变化时，编辑器会平滑地更新引用 ID 而不触发销毁，配合已有的「草稿转正识别逻辑」，确保保存过程丝滑无感。

2.  **设置页面自动下拉问题**：
    *   **原因定位**：日志终端使用了 `scrollIntoView`，这个方法会尝试让元素在整个窗口可见，从而拉动了父级滚动条。
    *   **修复方案**：改为通过操作容器的 `scrollTop` 属性实现精准滚动。现在日志只会悄悄地在自己的黑色方框内滚动，不再影响整个设置页面的布局。

3.  **更新功能增强与报错修复**：
    *   **报错修复**：修复了打包版（EXE）找不到 Git 路径的问题。现在程序能智能识别运行环境，准确定位 `.git` 文件夹和 `fast_update.bat` 脚本。
    *   **交互升级**：实现了你要求的「检查 -> 确认 -> 更新」三段式流程。
        *   **检查更新**：对比本地和远程 Commit Hash，告知是否有新版本。
        *   **确认更新**：发现新版本后需手动点击「确认更新」才会执行 `git pull`，避免意外覆盖。
        *   **状态反馈**：增加了「已是最新版本」、「发现新版本」、「操作失败（如未安装 Git）」等详细文案提示。
