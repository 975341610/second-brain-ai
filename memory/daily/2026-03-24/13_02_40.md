### 🐛 Bug 1【严重】：`update_note_api` 中 content 为空时会被吞掉
在 `routes.py` 第 437 行：
```python
content = payload.content or existing.content  # ← 有隐患！
```
`or` 会把**空字符串**（刚删光所有内容）视为 `False`，导致清空内容保存时，后端会把**旧内容**塞回去。正确写法是 `if ... is not None`。

---

### 🐛 Bug 2【严重】：自动保存有竞态条件（`isSaving` 锁的问题）
在 `NotionEditor.tsx` 第 449 行：
```tsx
if ((hasContentChanged || hasTitleChanged) && !isSaving) {
```
`isSaving` 是 React state，**异步更新**，不是同步锁。如果防抖刚好触发两次（1.5s 内连续改动），第二次 `isSaving` 还是 `false`，就会同时发出两个 PUT 请求，**后者可能覆盖前者**。

---

### 🐛 Bug 3【中等】：`useEffect` 依赖项包含了 `isSaving`，会引起循环
在 `NotionEditor.tsx` 第 477 行：
```tsx
}, [editor, note, onSave, isSaving]);
```
`isSaving` 变了 → `useEffect` 重新跑 → 重新注册 `update` 事件 → 每次保存都在抖动地"重装"监听器，还会把 `clearTimeout(timer)` 触发一次，导致**节流 timer 被意外清除**，引起保存遗漏。

---

### 🐛 Bug 4【中等】：500 报错来源 `restore_note` 的 typo
在 `repositories.py` 第 181 行：
```python
parent = db.get(Note, note_id=note.parent_id)  # ← note_id 是关键字参数 typo
```
正确写法是 `db.get(Note, note.parent_id)`，多了个 `note_id=` 关键字，会导致 **`restore_note` 接口 500 报错**。
