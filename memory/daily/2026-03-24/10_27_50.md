Agent: 哈哈，老大，GitHub Desktop 简直是“不想记 Git 命令”星人的救星！✨ 既然你都在用了，沫沫给你准备了一份**“三分钟极速上手秘籍”**：

### 📥 1. 把项目搬回家 (Clone)
*   **操作**：`File` -> `Clone Repository`。
*   **秘诀**：在列表里直接点你在 GitHub 上的项目，选好本地存放路径，点 `Clone`。代码就乖乖躺在你电脑里啦！

### 🌿 2. 开个小分身 (Branch)
*   **操作**：顶部中间的 `Current Branch` -> `New Branch`。
*   **建议**：改代码前先开个新分支（比如叫 `feat-new-ui`），这样万一写乱了也不影响主线代码。

### ✍️ 3. 记下你的改动 (Commit)
*   **操作**：你在编辑器（比如 VS Code）改完代码，左边列表会显示改了哪些文件。
*   **步骤**：
    1.  勾选你想提交的文件。
    2.  左下角 `Summary` 写个简短的总结（比如“修复了 chromadb 报错”）。
    3.  点那个大大的蓝色按钮 `Commit to [分支名]`。

### 🚀 4. 同步到云端 (Push / Pull)
*   **操作**：右上角的蓝色按钮。
*   **逻辑**：
    *   显示 `Publish branch`：第一次把新分支传上去。
    *   显示 `Push origin`：把本地改好的传上去。
    *   显示 `Pull origin`：把云端别人写的（或者你别的电脑写的）拉下来。

### 🤝 5. 合并代码 (Pull Request)
*   **操作**：代码 Push 上去后，点那个 `Create Pull Request` 按钮。
*   **结果**：它会帮你跳转到浏览器，点一下 `Create pull request`，你的改动就能合并进主分支（main）啦！

---

**💡 沫沫的小贴士：**
*   **改错想反悔？** 右键左侧列表的文件，选 `Discard Changes` 就能一键还原。
*   **看历史记录？** 点左上角的 `History` 标签，哪天改了啥清清楚楚！
