### 🛠️ 修复明细

#### 1. 笔记新建后显示异常 (Bug 1)
- **文件**: `frontend/src/store/useAppStore.ts`
- **原因**: 自动保存（`silent: true`）逻辑过于严格。当草稿（负数 ID）第一次保存变为正式笔记（正数 ID）时，Store 未同步更新 `selectedNoteId`，导致编辑器丢失引用。
- **修复**: 修改 `saveNote` 逻辑，确保“草稿转正”时无论是否静默保存都会修正 `selectedNoteId`。

#### 2. AI 功能无法使用 (Bug 2)
- **文件**: `backend/services/ai_client.py`
- **原因**: 所有的 AI 请求都强制设置了 `trust_env=False`，这会忽略系统代理。对于国内用户，这会导致无法连接到 OpenAI/Claude 等 API 终点。
- **修复**: 将 `trust_env` 改为 `True`，使应用能够识别并使用系统代理。
- **优化**: 统一了 `/ai/inline` 的返回格式为 `text/plain`，修复了后端声明为 SSE 但实际发送纯文本导致的格式不一致。

#### 3. 更新功能无法使用 (Bug 3)
- **文件**: `backend/api/routes.py`
- **原因**: 之前的 `git fetch` 缺乏超时控制，且 Windows 下的编码处理不健壮。
- **修复**:
    - 为 `git fetch` 增加了 15 秒超时锁定，防止网络不佳时接口卡死。
    - 强制使用 `UTF-8` 编码解析 Git 输出，避免 Windows 默认 GBK 导致的乱码。
    - 优化了错误提示，当未找到 `.git` 目录（如直接运行安装后的 EXE）时，明确提示用户。
