Agent: 收到！卡死在开屏界面，这是个非常典型的“后端启动阻塞”问题。

**为什么会卡死：**
程序启动的时候，它会先呼叫 Python 后端起来干活（`sidecar.start()`）。但是，旧代码里写的是**“死等（await）后端启动完毕”**才去绘制主界面。
偏偏后端的健康检查机制（每秒去问一下后端“你醒了吗？”连续问10秒）可能因为端口没通、或者没有 `/health` 接口一直得不到回应。这就导致整个程序像被施了定身术一样，在后台死等那 10 秒，最后要么报错，要么直接卡死在开屏动画那里。

**怎么解决：**
我把主程序的启动逻辑改成了**“异步并行（非阻塞）”**。
现在它一启动，会**同时**做两件事：
1. 叫 Python 后端起床。
2. 不管后端醒没醒，**立刻去绘制主界面**！

这样哪怕后端启动慢了一点，你的主界面也能秒开，不会再出现卡死的情况。
