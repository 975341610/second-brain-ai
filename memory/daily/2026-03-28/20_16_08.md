这其实是**“网页版”**和**“独立桌面软件版”**底层的本质区别：

1. **以前为什么两条命令就行？（MVP 阶段）**
   当你敲 `npm run dev` 的时候，其实你只是启动了一个“网页版”的前端皮肤。而此时，你本地电脑里是**自己装好了 Python 的**，而且你在后台自己把 Python 服务器给跑起来了。网页就像个客人，直接访问你跑好的 Python 服务，当然快，当然没问题。

2. **现在为什么 .exe 会卡死？（Electron 阶段）**
   我们要把它变成一个给**任何人（哪怕他电脑里没装 Python）都能直接双击运行的独立 .exe**。这就意味着，这个 `.exe` 必须把“前端皮肤”和“Python 灵魂（后端）”全都强行压缩进这一个文件里。
   
   **破案了！**我刚才查了一下代码发现：我们现在的云端打包流水线（GitHub Actions），**只打包了前端的皮，忘记把 Python 的后端灵魂打包塞进 `.exe` 里了！**
   
   所以当你双击运行 `.exe` 时：
   - 它的开屏动画出来了（皮起来了）。
   - 它在后台疯狂呼叫 Python 后端：“快醒醒，我要加载数据了！”
   - 但它发现 `.exe` 肚子里根本没有 Python 后端（灵魂走丢了）。
   - 于是前端就一直等，最后陷入了无限的卡死或白屏。
