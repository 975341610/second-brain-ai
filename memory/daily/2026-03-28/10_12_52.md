### 根因分析（为什么请求没有打到 8000 端口）：

1. **Python 后端根本就不在 8000 端口！**
   我刚刚查阅了 `backend/desktop.py`（以及咱们用来拉起后端的脚本），里面写死了：
   `port = int(os.environ.get("PORT", "8765"))`
   也就是说，当 Electron 的 `sidecar.ts` 拉起 Python 后端时，如果不传环境变量，**Python 默认监听的是 `8765` 端口，而不是 `8000`！**

2. **前端的 Proxy 代理配错了目标！**
   在刚才那次修复中，我理所当然地把前端 Vite 的 proxy 指向了 `http://localhost:8000`。结果就是：
   前端把请求发给 8000 端口 👉 8000 端口根本没人接听 👉 请求被退回，Vite Dev Server 把它当成非法路由，兜底返回了 `index.html` 👉 前端收到 HTML 报错 `Unexpected token '<'`。

### 怎么彻底根治？
要让两边对齐，必须统一端口：
1. **修改前端**：把 `electron-vite.config.ts` 里的 proxy 目标，从 `8000` 改为 `8765`。
2. **修改后端**：在 `sidecar.ts` 启动 Python 时，显式地注入环境变量 `PORT=8765`，确保它绝对不会乱飘。

这样才是正正规规打通全链路，而不是靠本地临时加个 `.env.local` 来逃课。
