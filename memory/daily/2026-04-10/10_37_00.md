Agent: 太棒了老大！无界画布（Infinite Canvas）这个功能简直是提升 Second Brain 维度的神级特性，做出来绝对有 Heptabase / Milanote 的那种高级质感和生产力！✨

为了保证**极致的性能**和**最优雅的代码架构**，在动手之前，我有几个技术实现上的细节想和您对齐一下：

### 1. 底层技术选型 (画布引擎)
要在 Web 端实现丝滑的无限画布、节点拖拽、自由连线、框选和缩放，从零手写非常耗费性能和时间。
👉 **我的建议**：引入目前 React 生态中最强大且性能极佳的 **`React Flow` (或 `@xyflow/react`)** 作为底层引擎。它原生支持自定义卡片节点、连线、框选编组（Group Nodes）以及缩放平移，完全符合您的需求。**您同意引入这个库吗？**

### 2. 数据库与存储设计 (白板=特殊笔记)
您提到：“白板和笔记一样，可以随意拖拽到任意文件夹内”。
👉 **我的建议**：为了完美复用现有的左侧目录树、拖拽移动逻辑，我们**不新建**单独的 `canvases` 表，而是直接在现有的 `notes` 数据库表里新增一个 `type` 字段（默认为 `note`，白板为 `canvas`）。白板的节点坐标、连线数据以 JSON 格式直接存在现有的 `content` 字段里。
这样改动最小、最稳定，且天然支持现有的文件夹体系！**这个数据设计思路您觉得 OK 吗？**

### 3. 卡片与归类交互设计
*   **节点卡片**：我会设计统一风格的卡片（笔记引用卡、图片卡、媒体卡），卡片内部提供一个可编辑的**文字备注区**。
*   **框选归类**：用户框选多个节点后，右键点击“编组/归类”，我会用一个带有标题的**大虚线框（Group Node）**将它们包裹起来，拖动大框里面的卡片也会跟着走。**这种交互形式符合您对“归类”的预期吗？**

### 4. UI 风格
我会强制启用 `ui-ux-pro-max` 规范，确保白板的背景（比如细腻的网格/点阵底纹）、连线（圆滑曲线）、卡片阴影（彩色弥散阴影）符合咱们整体的**治愈系/手账质感**，告别直男审美！
