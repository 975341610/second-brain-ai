### 🩺 病因分析

```
File "chromadb\api\models\CollectionCommon.py", line 112
    NameError: name 'ONNXMiniLM_L6_V2' is not defined
```

**根本原因**：
- ChromaDB 的默认向量嵌入函数（`ONNXMiniLM_L6_V2`）依赖 `onnxruntime` 这个大型库。
- PyInstaller 打包时，`onnxruntime` 没能被正确识别为动态依赖，导致符号表里压根没有这个类。
- ChromaDB 在创建 Collection 时默认会用它，结果打包后一调用就炸了！

---

### 🛠️ 我已经做了双重修复：

**修复一（`vector_store.py`）—— 釜底抽薪 🔥**
> 定义了一个轻量级 `NoEmbeddingFunction` 并显式传给 ChromaDB。这样 ChromaDB 再也不会去找那个"不存在的" `ONNXMiniLM_L6_V2` 了。

原因：我们项目里向量嵌入是前端自己做的，后端根本不需要让 ChromaDB 内置的嵌入器工作，所以这样做完全正确，不影响任何功能。

**修复二（`desktop.py`）—— 安全兜底 🛡️**
> 在 app 加载之前，先检查 `ONNXMiniLM_L6_V2` 是否存在。如果没有（打包环境里就是没有），就注入一个空的 Mock 对象，让程序不会在启动时就崩溃。
