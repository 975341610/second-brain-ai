这个 `NotFoundError` 是典型的 **DOM 争抢**问题：React 试图在编辑器内部更新我们那个“行列添加”按钮，但此时 Tiptap (ProseMirror) 可能正在重构表格结构，导致 React 找不到原本的 DOM 节点而崩溃。

**🔧 沫沫刚才做了这两项“外科手术”：**

1.  **彻底隔离 (React Portal)**：我把原本放在编辑器里的悬浮按钮容器，通过 `Portal` 技术直接挂载到了 `document.body` 最外层。这样无论编辑器内部怎么翻江倒海，都不会干扰到 React 的 DOM 追踪逻辑。
2.  **生命周期加固**：在更新表格坐标时增加了更严格的挂载状态检查，防止组件在销毁边缘时依然强行更新。

**现在的逻辑：**
- 编辑器只管渲染内容。
- 悬浮按钮像“投影”一样根据坐标定位在表格上方，两者物理隔离，互不干扰。
