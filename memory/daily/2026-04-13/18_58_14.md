Agent: 老大，关于“块手柄严重漂移”的 Bug，我已经揪出罪魁祸首并修复啦！✨

### 🔍 案案侦破过程 (Root Cause)
经过对 Tiptap 拖拽手柄底层 `@floating-ui/dom` 源码的追踪，发现根本不是页面滚动的相对坐标问题，而是 **“垂直居中对齐”导致的视觉漂移**！
- Tiptap 的官方拖拽手柄组件默认使用了 `{ placement: 'left' }`（纯左侧对齐）。
- 这意味着，手柄会永远对齐到当前 Block 的**垂直中心**。当你在一行里敲字，段落不断换行变高时，这个手柄就会不断**往下方漂移**（始终保持在多行的正中间）。如果是一个几百像素高的大图或长段落，手柄甚至会跑到底下好长一截，完全偏离了我们预期的“首行左上角”，给你造成了“严重漂移”和错位的错觉！

### 🛠️ 解决方案 (The Fix)
我在 `NovaBlockEditor.tsx` 的 `<DragHandle>` 组件中强行覆盖了底层的定位计算规则：
```tsx
// 引入 computePositionConfig 强制吸顶
computePositionConfig={{
  placement: 'left-start'
}}
```
现在，它和 Notion 一模一样了，死死地锚定在每个 Block 渲染盒子的**左上角（首行位置）**，不管你输入多少行，它的垂直高度雷打不动！
