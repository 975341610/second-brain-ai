Agent: **绝对支持！** 而且咱们现在的底层设计，就是“原生为了兼容中转网关”而打造的。

咱们的笔记软件在 AI 接入上，遵循的是**“全宇宙通用的 OpenAI 兼容协议”**。

具体来说，只要你的非官方中转平台（比如 NewAPI、OneAPI 搭建的中转站，或者是那种几毛钱 1M Token 的第三方聚合站）**支持 OpenAI 的 `/v1/chat/completions` 接口标准**，咱们全都能完美对接。

### 🔌 我们目前为了兼容中转网关做了这几层“特权设计”：
1. **强制忽略 SSL 证书验证 (`verify=False`)**：很多便宜的中转站用的是不标准的证书，或者你本地挂了特殊的代理工具，普通的客户端直接就报错不干了。咱们的软件直接无视证书错误，强行打通。
2. **自动补全路径 (`/v1`)**：中转站的地址五花八门，有带后缀的、没带后缀的。只要你填进去，我们会在发请求前自动规整，哪怕你忘了写 `/v1`，咱们也会试着帮你补上。
3. **识别 Cloudflare / Nginx 拦截**：很多中转站外面套了一层防护（比如防御被 DDoS 攻击），万一他们挂了返回一个死板的 HTML 页面，咱们的软件不会直接崩掉，而是会自动翻译成“网关源站不可达”。

### 🛠️ 你要怎么填？
你在中转站拿到配置后，只需要在咱们的【设置 - 模型配置】里填这两样：
- **Base URL (接口地址)**：填他们给的地址（通常长这样：`https://api.xxxx.com/v1`）
- **API Key**：填他们在后台给你生成的令牌（通常是 `sk-` 开头）
- **模型名称**：填你想调用的那个名字（比如 `gpt-4o`, `claude-3-5-sonnet-20240620` 等等，只要他们支持就行）

所以老大，你刚才那个 `https://gpt-agent.cc` 其实也是能用的，只是当时它恰好触发了 522（后端服务器宕机），只要它的服务器一恢复，你在咱们这里立刻就能满血跑起来！🚀
