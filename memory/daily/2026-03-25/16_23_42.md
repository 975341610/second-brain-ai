1) **表格按钮只有“光标在表格里”才生效**
对，这就是典型的 ProseMirror/Tiptap 命令依赖 `state.selection` 的问题：你鼠标点按钮时，编辑器可能已经失焦或 selection 不在 table 里，命令就被判定为不可执行。
解决思路是：**按钮点击时强制把 selection 修复到“按钮对应的那张表/那一行”**，再执行 add/delete。这样就不再依赖“鼠标必须先点进表格”。

2) **AI 报错：All connection attempts failed**
这不是协议层（Anthropic vs OpenAI）的问题，这个报错通常发生在更底层：
- 域名/端口连不上（被墙/代理/VPN/公司网络策略）
- base_url 指到一个不可达地址
- TLS/证书/代理导致握手失败

但你提到 Anthropic 协议也很关键：如果你填的是 Anthropic 的 endpoint，我们目前很可能按 OpenAI 的 `/v1/chat/completions` 去请求，服务端会返回 404/401/非 200；不过那种一般不是“connection attempts failed”，而是有 HTTP 状态码。
