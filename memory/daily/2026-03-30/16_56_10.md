**1. 妖精一：编辑器状态泄露（导致内容被覆盖）**
当您在父页面写得正欢，突然点击“新建子页面”时，父页面那个“1.5秒后自动保存”的定时器还在倒数！等时间一到，它突然发现：“咦？当前笔记变成了子页面！”于是，**它居然丧心病狂地把父页面的内容，直接保存到了子页面的头上（或者反过来）！** 导致您一回头，父页面内容没了或者乱套了。
👉 **怎么镇压的**：代码小助手现在给防抖保存加了一把**锁（ID 锁定机制）**！每次触发保存，它都会死死盯住触发那一刻的笔记 ID，即使您中途切到了别的页面，它也只会把数据稳稳地存回原来的那个 ID 里！

**2. 妖精二：状态引用污染（导致侧边栏结构错乱）**
在前端的状态管理库里，当新建一个子页面并把它挂到父页面下面时，之前不小心用了**“直接修改”（直接 push）**的粗暴手法，没有老老实实地创建“新的对象替身”（不可变状态更新）。这就导致 React 框架彻底眼瞎了，它根本没发现父页面被更新了，导致显示直接串位或者卡住！
👉 **怎么镇压的**：已经全面改写成了标准的不可变数据更新逻辑（Immutable Update），现在 React 只要一看到新对象，马上就能精准渲染出完美的树形结构！
