原来，在笔记从“草稿”变成“正式”的那一瞬间，如果本地列表里恰好因为缓存等原因存在同 ID 的数据，前端的状态合并逻辑竟然会**直接把新笔记的内容覆盖到旧笔记的头上**！这就导致旧笔记看起来像是变异了一样，而且还会引发 React 渲染引擎的 Key 冲突崩溃。

**2. 为什么子页面像牛皮癣一样删不掉？（孤儿节点 Bug）**
