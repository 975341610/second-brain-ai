Agent: 老大的直觉非常敏锐！在富文本编辑器里加特效，绝对符合我们想要打造的“酷炫/游戏化（The Cool Redesign）”视觉体验，但如果做不好，确实会变成性能杀手，导致滚动掉帧、笔记本发热。

不过别担心，我们可以用**“性能优先的架构”**来完美平衡酷炫与流畅！针对这个问题，我建议采用以下 **4 道性能保险** 来实现这些特效：

### 1. 纯 CSS 硬件加速 (GPU Acceleration)
**坚决不写一行 JS 动画！** 所有的动画（打字机、渐变、跳动）全部写成纯 CSS 的 `@keyframes`。
- **跳动/跑马灯**：只使用 `transform: translateY()` 或 `translateX()`。这在浏览器底层是直接交给 GPU 复合层处理的，**完全不触发页面的重排 (Layout) 和重绘 (Paint)**，性能开销极低。
- **动态渐变 (Animated Gradient)**：使用 `background-clip: text` 配合 `background-position` 动画，这也是目前非常成熟且高性能的文字渐变方案。

### 2. 视口按需播放 (Intersection Observer / CSS `content-visibility`) —— 最核心的杀手锏！
这是解决长文档性能崩盘的关键：**不在屏幕里的东西，坚决不让它动！**
我们可以给特效文字加上简单的可见性控制，当含有特效的文字滚动出屏幕外时，动画自动冻结/暂停；只有滚动到眼前时才播放。这样哪怕你的一篇笔记里塞了 100 个跑马灯，真正在消耗 GPU 的也只有屏幕当前显示的那 2 个。

### 3. 细粒度控制 (作为 Tiptap Mark 而非 Block)
我们会把这些特效做成像“加粗”、“斜体”一样的**局部标记 (Mark)**，而不是整个段落。用户只会对几个关键的词语（比如标题、特定名词）加上 `✨ 动态渐变` 或 `💬 打字机` 特效，避免了大面积文字同时做动画的灾难。

### 4. 全局“性能/省电模式”开关
为了应对低配轻薄本或电池模式，我们可以留一个全局的 CSS 变量。用户在设置里一键开启“禅定模式/省电模式”，所有文字特效瞬间变成静态的漂亮渐变色，彻底归零性能消耗。

只要我们严格遵守 **纯 CSS 硬件加速 + 视口外冻结** 的铁律，加入这几种文字特效对现在的应用来说性能损耗微乎其微！不仅不会卡，反而会让笔记看起来有一种高级的“赛博朋克”或“Notion Pro Max”的质感。
