Agent: 老大，这个报错 `Could not resolve host: github.com` 是因为你的电脑当前无法解析 GitHub 的域名，也就是网络暂时连不上 GitHub 啦！(T_T) 毕竟你在重庆，国内直连 GitHub 经常会遇到这种“间歇性失联”的 DNS 污染问题。

别着急，这是小场面，咱们用下面这几招就能搞定：

**方法一：检查代理并配置 Git（最常见的情况）**
如果你电脑上开着代理（梯子，比如 Clash、V2Ray 等），Git 默认是不会乖乖走代理的。你需要打开终端（CMD 或 PowerShell），告诉 Git 你的代理地址。
（假设你的代理端口是常见的 `7890`，如果不是的话可以去软件设置里看一眼本地端口）：
```bash
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890
```
敲完这两行，再试一次 `git pull` 应该就畅通无阻啦！✨
*(以后如果关了代理想取消，可以敲这俩：`git config --global --unset http.proxy` 和 `git config --global --unset https.proxy`)*

**方法二：刷新一下 DNS 缓存**
有时候是系统本身的 DNS 缓存卡住了，可以在终端里敲这行命令清理一下：
```bash
ipconfig /flushdns
```
清理完再试着 pull 一下。

**方法三：使用神奇的加速工具**
如果你现在没开代理，非常推荐开一下 **Watt Toolkit (原 Steam++)** 并勾选 GitHub 加速，免费又好用，能完美解决连不上 GitHub 的痛点！
