Agent: 老大，这就给您系统地梳理一遍咱们目前的**全新架构（Local-First SSOT）**以及**后续的演进路线**。

之前的“前端 + FastAPI + 各种接口存 SQLite”架构，因为数据同步太依赖网络和状态管理，导致了无数的“保存失败”、“草稿冲突”和卡顿。所以咱们现在彻底转向了类似 Obsidian / Typora 的**纯本地文件优先架构**。

---

### 一、当前架构：纯本地优先 (Local-First SSOT)

**核心思想：**
**“本地文件即唯一真理 (Single Source of Truth, SSOT)”。** 一切以您电脑磁盘上的 `.md` 文件为准，不依赖任何网络请求，也不依赖中间数据库的草稿状态。

**架构四要素：**

1.  **物理存储层 (The File System)**
    *   **位置：** 您电脑上的本地文件夹（Windows 下通常在 `AppData` 或您指定的目录）。
    *   **格式：** 纯文本 Markdown 文件（带 YAML Frontmatter 保存标签和状态）。
    *   **回收站：** 本地 `.trash` 文件夹（软删除，物理隔离，不依赖网络）。

2.  **主进程调度层 (Electron Main - Node.js)**
    *   **FS Watcher：** 像雷达一样死死盯着本地 `.md` 文件。如果您用 Obsidian 改了文件，主进程毫秒级捕获，并推给前端。
    *   **Python 常驻桥接：** 将原本需要走 HTTP 的后端逻辑（比如复杂的文本解析、向量检索），降级为通过本地标准输入/输出或常驻子进程（IPC Bridge）进行极速调用，**杜绝网络超时，实现“毫秒级秒存”**。

3.  **前端渲染层 (Electron Renderer - React)**
    *   **完全断网运行：** 去掉了所有 `fetch(http://localhost:8000)` 的网络请求，所有增删改查通过 `window.electron.ipcRenderer` 直接使唤底层主进程。
    *   **离线缓存：** 引入了 `IndexedDB`。哪怕您拔了网线、主进程还没反应过来，前端也能瞬间把上次编辑的内容从浏览器本地缓存里刷出来（秒开）。

4.  **协同冲突合并层 (CRDT - Yjs + Tiptap)**
    *   为了防止多开窗口或外部修改导致的冲突，引入了 `Yjs`（无冲突复制数据类型）。它能像 Git 合并代码一样，把前端的修改和本地文件的修改静默合并，这也是为什么我们刚才花大力气装那些协作插件的原因。

### 二、我们的演进路线 (The Roadmap)

现在我们正处在“进阶升级蓝图”的**第一阶段（MVP）收尾期**。路线如下：

#### 📍 里程碑 1：本地优先 MVP（正在攻克）
*   **目标：** 让基础的记笔记（新建、编辑、重命名、删除）彻底摆脱网络，变成毫秒级响应，且不丢数据。
*   **进度：** 底层 IPC 已打通，保存失败/闪烁等 Bug 已修复。当前卡点在于**前端协作依赖包的安装冲突**。
*   **下一步动作：** 等您本地环境把 `Yjs` 和 `Tiptap-collaboration` 彻底跑通，能丝滑体验断网秒存。

#### 📍 里程碑 2：多端同步体系 (Sync)
*   既然文件都在本地，下一步就是把这些本地文件同步到其他设备（手机、另一台电脑）。
*   我们将引入基于 Git 或者增量同步方案（配合刚才引入的 CRDT 冲突合并机制），让它像坚果云或 iCloud 一样在后台静默同步。

#### 📍 里程碑 3：向量检索与 AI 重连 (AI RAG)
*   基础读写稳了之后，我们要把之前拆掉的 Python AI 能力通过纯本地通道接回来。
*   本地的 Markdown 文件变动会触发主进程在后台偷偷更新本地向量库（ChromaDB），实现本地的智能问答和“第二大脑”关联。

#### 📍 里程碑 4：完全体与商业化 UI (The Polish)
*   整合所有的扩展功能：比如外网安全访问（Tailscale）、二次元交互（看板娘联动）。
*   进行全站 UI 重构（毛玻璃特效、沉浸式窗口等），把目前的套壳网页彻底变成一个原生质感的独立桌面端。
