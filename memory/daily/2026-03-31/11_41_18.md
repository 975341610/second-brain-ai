咱们现在的架构是 Electron (主进程) + React (渲染进程)，需要两边都启动。请在您本地的代码编辑器终端（比如 VSCode 的 Terminal）里，依次运行下面这几步：

### 第 1 步：确保依赖是最新的（非常重要）
因为我们加了新的本地存储包（像 `y-indexeddb` 和 `chokidar`），所以得先重新安装一下依赖：
```bash
npm install
```
*(如果网络慢或者报错，您可以试试用 `cnpm install` 或 `yarn install` 替代，平时您用哪个包管理器顺手就用哪个)*

### 第 2 步：一键启动！🚀
依赖安装完后，输入咱们平时最常用的启动命令：
```bash
npm run dev
```
*(这个命令会自动帮您把 React 前端跑起来，同时拉起 Electron 的独立窗口)*

### 第 3 步：开始体验！😎
等界面出来后，您可以按照下面的姿势“蹂躏”它：

1. **测试秒开**：关掉网络，或者直接重启软件，看看内容是不是瞬间就加载出来了？（这就是 IndexedDB 的功劳）
2. **测试外部修改同步（重头戏）**：
   - 随便找一篇您的笔记，右键选“在文件夹中显示”或用系统文件管理器打开它对应的 `.md` 文件。
   - 用 Obsidian 或 Typora 打开这个 `.md` 文件，在里面随便敲几个字并保存。
   - 切回到咱们的软件，看看刚才敲的字是不是**瞬间自动出现**了？（这就是主进程 Watcher 监视本地文件的威力！）
3. **测试防丢机制**：随便敲点内容，直接点右上角的“X”关掉软件，再从系统托盘里双击恢复，看看刚才敲的内容还在不在？
