任务描述：
请为我的笔记应用开发一个位于页面右侧的目录（TOC）组件。该组件需要具备高度的视觉美感和极简主义风格，重点在于从“抽象横线”到“具体文字”的丝滑过渡。
核心功能需求：
 * 初始状态（静止）：
   * 固定在页面右侧（Fixed position）。
   * 不显示文字，仅根据标题等级（H1, H2, H3...）显示长短不一的水平横线。
   * 例如：H1 最长，H2 次之，H3 最短。横线颜色建议为淡灰色，保持极低的存在感。
   * 整个组件宽度极窄（如 20px - 40px），不占用正文视觉空间。
 * 交互状态（悬停 Hover）：
   * 当鼠标悬停在目录区域或特定行时，目录平滑向左展开或在原位显示对应的标题文字。
   * 文字应在横线位置淡入，横线消失或转化为文字的底纹。
   * 动效要求： 使用 transition 或 framer-motion 实现宽度、透明度和位移的补间动画，曲线要求 ease-in-out，时长约 0.3s。
 * 功能逻辑：
   * 同步高亮： 当页面滚动时，对应当前阅读位置的横线/文字需要有高亮颜色（如主题色）。
   * 点击跳转： 点击目录项平滑滚动（Smooth Scroll）到正文对应位置。
   * 自动生成： 自动提取正文中的 <h1> 到 <h3> 标签并构建层级树。
技术栈建议：
 * 请使用 React/Vue 组件化开发。
 * 样式推荐使用 Tailwind CSS 处理基础布局，结合 CSS Modules 或 Styled-components 处理复杂的动效逻辑。
 * 确保在悬停切换时，文字不会因为宽度突变而产生抖动（建议使用 white-space: nowrap 和固定最大宽度）。
视觉参考：
想象它像是一个垂直的“条形码”，当你靠近它时，它才优雅地向你展示其蕴含的信息。
💡 给开发者的额外建议（开发技巧）
在实现这个功能时，有几个细节可以让体验更好：
 * 占位宽度技巧： 为了防止目录展开时挤压左侧正文，建议目录使用 position: fixed 或 absolute。如果希望正文自动避让，可以使用 grid-template-columns: 1fr auto; 并通过动画改变 auto 列的 min-width。
 * 层级缩进： 即使在横线模式下，也应该保留微小的左侧 padding-left 或 margin-left 偏移，这样用户通过横线的错位就能一眼看出文档的逻辑结构。
 * 贝塞尔曲线： 推荐使用 cubic-bezier(0.4, 0, 0.2, 1)，这会让展开动作看起来更有“高级感”。
