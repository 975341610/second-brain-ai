# v0.5.0 System Fix Design Document

Based on P0/P1 audit findings for `second-brain-ai` project.

## 1. DB Session & Lifecycle Governance
- **Issue**: Background tasks (`background_index_note`) use a closed DB session after the initial `with` block, causing `Session closed` errors.
- **Fix**: Re-structure `background_index_note` to manage the DB session lifecycle properly. The session will remain open for the duration of the background task and closed in a `finally` block.
- **Implementation**:
  ```python
  async def background_index_note(...):
      db = SessionLocal()
      try:
          # perform operations
          db.commit()
      except Exception as e:
          db.rollback()
          raise e
      finally:
          db.close()
  ```

## 2. Streaming Optimization
- **Issue**: Buffer backlog causes laggy AI responses. SSE parsing is not robust.
- **Backend Fix**:
  - Enhance `AIClient.stream_chat` SSE parser to be more tolerant (strip lines, handle empty lines, robust prefix check).
  - Update `StreamingResponse` in `routes.py` with headers: `X-Accel-Buffering: no`, `Cache-Control: no-cache`.
- **Frontend Fix**: (Currently fetch reader is fine, but backend headers will improve latency).

## 3. Performance Hardening
- **HTTPX Connection Pool**:
  - Configure `httpx.Limits` in `AIClient` to reuse connections efficiently and handle concurrency.
- **DB Lock Retry**:
  - Implement a retry mechanism for database writes (especially in background tasks) to handle SQLite `OperationalError: database is locked` during high concurrency.

- **Load Test**: Verify streaming performance under simulated lag.
- **Concurrency Test**: Ensure multiple background tasks succeed without session errors.
