# 待办/后续计划 (优先级调后)
- **Phase 4: 桌面究极体改造 (Electron化)**: 弃用 `PyWebView` 和 `系统默认浏览器` 的过渡方案，引入 Electron 框架进行重构，实现类似 Obsidian/Notion 的商业级独立桌面窗口，支持无边框窗口、原生系统托盘(Tray)驻留、更稳定的全局快捷键接管，并彻底解决环境依赖打包问题。
- **全站 UI & 布局重构 (The "Cool" Redesign)**: 摆脱平庸，打造酷炫视觉体系，参考 Awwwards、Linear、Apple、Reflect。待功能补齐后再执行。
- **飞书级表格系统**: 增强 Tiptap 表格的多维属性能力。
- **万物皆可拖拽 (Drag & Drop)**: 支持文档/文本/表格/图片/视频/音频的直接拖拽，支持自由调整位置和大小。
- **互动级动态壁纸系统 (Wallpaper Engine 级别)**: 将当前静态视频壁纸升级为动态可交互体系：(1) 环境感知（随系统时间、天气自动无缝切换早中晚场景）；(2) WebGL/Three.js 交互特效（鼠标水波纹、打字震动、粒子视差）；(3) 创意工坊与看板娘整合（支持 `.glsl` 着色器代码直读与 Live2D 互动结合）。此功能将作为后期重点体验升级打包进大招池。

- **chromadb 打包终极修复 (v0.3.36)**: 彻底弃用了不稳定的模块注入/劫持方案（该方案会引发 `partially initialized module` 循环导入报错）。
    - **懒加载 (Lazy Import)**: 重构了 `backend/services/vector_store.py`，使 `chromadb` 及其所有依赖项仅在第一次数据库操作时（`upsert/search`）才被载入。
    - **单例模式**: 确保 `vector_store` 在模块级别不触发任何 chromadb 导入操作，彻底从物理层面上切断了启动时的循环导入链路。
    - **兜底注入**: 在 `desktop.py` 中仅保留极简的 `builtins` 注入，作为 ONNX 模型缺失时的安全降级。
- **极速保存**: 后端全异步处理（BackgroundTasks），保存秒回。
- **编辑器防抖**: 1.5s 停顿触发保存，不再频繁刷屏。
- **草稿转正修复**: 解决新建笔记 ID 变化导致的内容重置（Draft Promotion Fix）。
- **表格 UI 优化**: 隐藏属性标签，优化操作菜单。
- **代码块深度定制**: Mac 风格三色灯、自动高亮、复制代码。
